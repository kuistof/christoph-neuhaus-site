AI-Agenten (Claude API, .md-Files)

Jeder Agent ist eine Markdown-Datei mit eigenem Kontext, Scope und Tonalität. Der Triage-Agent ist der Router — er erkennt das Projekt und leitet an den richtigen Agenten weiter.

triage-agent.md — Router
Scope: Alle Kanäle. Ordnet Nachrichten dem richtigen Projekt zu. Schätzt Priorität. Schreibt Auto-Acknowledgments.
Kennt: Alle Projekte, alle Postfächer, alle aktiven Deals, alle Agenten-Scopes.

n8n-Workflows (Mac Mini, Docker)

Alle Workflows folgen dem Prinzip: Human-in-the-Loop — Agenten schlagen vor, Christoph bestätigt. Kein Agent tritt eigenständig in Kundenkontakt.

1
E-Mail-Triage & Erstreaktion
Trigger: IMAP (4 Postfächer, alle 5 min)
IMAP Node → Claude: Triage → Projekt-Agent → Supabase Write → WA Notification → HITL → SMTP senden
Ziel: Nie wieder E-Mail übersehen. Erstreaktion innerhalb von Stunden.
2
WhatsApp auto-acknowledge & triage
Trigger: WhatsApp Business API Webhook
WA Webhook → Auto-Reply ✓ → Claude: Triage → Supabase Task → HITL
Voraussetzung: WhatsApp Business API + Webhook-Endpoint. Höchste Setup-Hürde.
3
Meeting → Action Items → Tasks
Trigger: Transkript-Upload (manuell oder Webhook z.B. Otter.ai, Fathom)
File Upload → Projekt-Agent → Action Items → Drafts in DB → HITL
Ziel: Nach jedem Meeting sofort alle Follow-ups als Draft.
4
Morning Brief (täglich 08:00)
Trigger: Cron, 08:00 Uhr
Cron → IMAP Scan → GCal API → Supabase Query → Claude: Briefing → WA an Christoph
Inhalt: Offene Tasks, heutiger Kalender, überfällige Antworten, Prioritäten.

~
TLW Admin vs. Beratung: Beides Ops-intensiv, beides braucht Fokus — direkte Zeitkonkurrenz.
~
Döring PM-Drift: Wird in PM/Asset-Erstellung reingezogen statt nur Sales + Strategy.
—
2 Beratungsadressen: @panamono.com vs. @christoph-neuhaus.de — Infrastruktur-Schulden, kein GWorkspace.
—
Tool-Wildwuchs: Pipedrive + Monday + Notion + Asana + GDrive + 4 Postfächer. Kein Single Source of Truth.
